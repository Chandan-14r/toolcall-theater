🤖 Specialized Agent Roles

The platform assigns tasks to specialized roles with strict tool execution limits and capabilities.

🛡️ Coordinator

Decomposes user tasks into plan steps. Schedules handoffs between roles and triggers rework loops if verification fails.

🔍 Researcher (Alice)

Focuses on information retrieval. Utilizes websearch and browser tools. Restricted to read-only filesystem scopes.

💻 Programmer (Bob)

Responsible for writing, updating, and executing code. Utilizes filesystem, shell, and python sandboxes.

✨ Reviewer (Charlie)

Validates outcomes. Run checks on code changes, runs tests via shell scripts, and evaluates final results against the brief.

🛠️ Sandboxed Tool Registry

Tools are executed inside restricted environments to prevent path traversals and unauthorized access.

🌐 Web Search (Tavily)

Retrieves high-quality pages from the live web. Requires TAVILY_API_KEY.

🦊 Browser (Playwright)

Automates chromium instances, performs user interaction simulations (clicking, filling forms), and captures step screenshots.

📁 FileSystem Sandbox

Grants read and write operations restricted strictly inside the local sandbox/ directory.

🐍 Python Sandbox

Evaluates mathematical operations and data processing scripts in an isolated subprocess.

🐚 Shell Executor

Runs command line processes inside the project's sandboxed root folder.

🧠 Hybrid Persistent Memory

The platform implements two complementary memory architectures for ultimate durability and semantic recall.

📊 SQLite Relational Schema

Stores structured run histories, agent step logs, tool invocations, durations, and outputs with full foreign key constraints and cascade delete operations.

🌐 Vector Space Model (VSM)

A zero-dependency TF-IDF vector database. Indexes planned tasks and their outcomes. Performs cosine similarity queries during Coordinator planning to recall past matching sessions.
